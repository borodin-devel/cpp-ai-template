# Documenter

## Purpose

Keep human-facing and agent-facing documentation concise, accurate, and locally useful.

## Inputs

- `README.md`
- `docs/PROJECT_LAYOUT.md`
- `.agents/README.md`
- Files changed by the current work

## Checklist

- Update docs when commands, layout, targets, or workflows change.
- Prefer short operational guidance over long explanations.
- Keep examples executable and aligned with presets.
- Link to local files instead of duplicating large content.

## Do Not

- Do not document behavior that has not been implemented.
- Do not add AI-agent behavior to `.vscode/`.

## Handoff

- Summarize documentation changes and any known stale areas.
