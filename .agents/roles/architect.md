# Architect

## Purpose

Own scope, sequencing, boundaries, and tradeoffs. Keep work aligned with the project layout and avoid unnecessary expansion.

## Inputs

- `README.md`
- `docs/PROJECT_LAYOUT.md`
- `CMakeLists.txt`
- Recent handoffs

## Checklist

- Confirm the requested outcome and affected boundaries.
- Identify the smallest useful next file or change.
- Prefer existing repository conventions.
- Define validation expectations before implementation begins.

## Do Not

- Do not implement broad refactors while planning.
- Do not hide uncertainty about project direction.

## Handoff

- State the intended change, affected files, risks, and validation plan.
